The user wants to create a comprehensive financial application for Android that functions as a modern Point of Sale (POS) terminal.

The core requirements, as clarified and updated throughout the conversation, are:
1.  **Smart Card Payments:** The application's primary function is to accept payments by debiting custom JCOP/Javacard smart cards ( chip).
    *   The connection to the card reader () must be handled via **USB-C only**. Bluetooth functionality was requested to be removed.
    *   The app must handle different card types:
        *   **Programmed JCOP Card:** Detect the custom applet and proceed with the transaction.

        *   **Standard Bank Card (Visa/Mastercard):** Reject the card as non-compliant, without attempting to read sensitive data.
2.  **Merchant Wallet Configuration:**
    *   On first launch, the application must prompt the merchant to enter their crypto wallet address (e.g., from Metamask).
    *   The application must validate this address in real-time on the blockchain (supporting both BSC and Ethereum, with BSC as default) and display the wallet's balance and transaction count. This address is where the funds from card payments will be sent.
3.  **Payment Flow:**
    *   The merchant sets their receiving wallet address.
    *   Home screen shows balances and navigation.
    *   New Payment: Choose currency (XAF, EUROM, TND) -> Enter amount -> Connect to card reader via USB -> Insert card -> App validates card type -> Card signs the transaction -> App confirms debit.
    *   The Crypto Link payment method has been explicitly **removed**.
